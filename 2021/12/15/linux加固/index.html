安全等级划分

等级划分 安全性
D级 最低安全性
C1级 主存取控制
C2级 较完善的自主存取控制（DAC），审计
B1级 强制存取控制（MAC）
B2级 良好的结构化设计，形式化安全模型
B3级 全面的访问控制，可信恢复
A1级 形式化认证

了解Linux中的文件

查看Linux的相关信息

cat /proc/version

| /bin | 放置的是在单人维护模式下能被操作的指令，在/bin底下的指令可以被root与一般账号所使用 |
| /boot | 这个目录只要在放置开机会使用到的文件，包括 Linux核心文件以及开机选单与开机所需配置的文件等等 |
| /dev | 在Linux系统上，任何装置与接口设备都是以文件的形态存在于这个目录当中的 |
| /etc |

系统主要的配置文件几乎都放在这个目录内，例如人员账号密码各种服务的启动档，系统变量配置等

| /home | 这个是系统默认的用户家目录（home directory) |
| /lib | /lib放置的则是在开机时会用到的函式库，以及在/lib或/sbin底下的指令会呼叫的函式库 |
| /media | /media底下放置的是可以移出的装置，包括软盘、光盘、DVD等等装置都挂载于此 |
| /opt | 给第三方协议软件放置的目录 |
| /root | 系统管理员（root）的家目录 |
| /sbin | 放置/sbin底下的为开机过程中所需要的，里面包括了开机、修复、还原系统所需的指令。 |
| /srv | srv可视为[service]的缩写，是一些网络服务启动之后，这些服务所需要取用的数据目录 |
| /tmp | 这是让一般使用者或是正在执行的程序暂时放置文件的地方 |

密码策略

什么是密码策略？

用于确定新密码是否有效的密码强度规则

什么是密码强度？

密码强度规则是密码必须符合的规则

例：

密码的最小字符数为5

修改密码策略的标准与规则

  • 最小和最大长度
  • 字符限制
  • 密码复用频率
  • 不允许使用的用户名或用户标识
  • 指定最短密码寿命

修改方法（Linux）

查看口令生存期

gedit /etc/login.defs

进入/etc/login.defs文件下进行修改

最常用的
PASS_MAX_DAYS   99999　　#密码最大有效期
PASS_MIN_DAYS   0 　　  #两次修改密码的最小间隔时间
PASS_MIN_LEN    5     #密码最小长度，对于root无效
PASS_MAX_LEN         #密码最大长度，对于root无效
PASS_WARN_AGE   7      #密码过期前多少天开始提示
passwd命令   root用户运行passwd ，可以设置或修改任何用户的密码

passwd命令：

查看修改结果

chage -l account

口令复杂性设置

1.安装cracklib， 能提供额外的密码检查能力

进入/etc/pam.d/common-password或/etc/pam.d/system-auth进行编辑

retry=5      定义登录、或修改密码失败后，可以重复的次数
type=        选项，定义用户修改密码时的提示内容，一般不管
dcredit=-1   至少一个数字
lcredit=-1   至少一个小写
ucredit=-1   至少一个大写
ocredit=-1   至少一个特殊字符
minlen=8     密码最短长度为5位
remember=5   不能重复使用前5次使用过的密码

修改方式

1.命令

password [success=1 default=ignore] pam_unix.so obscure sha512 remember=5                                                                         --Debian、Ubuntu 或 Linux Mint 系统
sudo vi /etc/pam.d/system-authpassword sufficient pamunix.so sha512 shadow nullok tryfirstpass useauthtok remember=5
                                                               ---CentOS、Fedora、RHEL 系统

2.vim进入文件进行修改

练习

1.密码最大有效期90天，过期前10天提醒用户更改密码

2.禁止使用旧密码

3.密码必须至少包含一个大写字母（ucredit），两个小写字母（lcredit），一个数字（dcredit）和一个标点符号（ocredit）

4.密码长度在7到12之间

5.密码长度在6为以上，密码中必须有一个大写字母

登录策略

什么是登录策略？

允许您控制用户在何时从何处登录到系统

提供实施登录活动相关策略的能力

控制用户访问的基本机制有哪些？

  • 对用户定义每日登录时间限制，而与他们从何处登录无关

  • 定义本地和远程机器上的访问控制

每日登录时间限制

定义允许用户在每周星期几、每天几点钟期间登录

登录次数限制

定义允许用户登录的次数及锁定的时间

文件：etc/pam.d/system-auth
unlock_time 单位秒
pam_tally2.so deny  尝试次数
假日登录限制

假日来指定额外的每 日时间限制

指定重复假日

假日日期扩展属性指定多个值来指定重复假日

登录位置限制

指定用户可以登录的位置

主要文件

1./etc/issue   用于显示欢迎信息 vim编辑

检验方式：init 3     本地
2./etc/ssh/sshd_config ssh配置文件 远程登录

远程用户非活动会话连接超时适应

进入文件

vi /etc/ssh/sshd_config 
找到ClientAliveInterval 进行修改单位秒
ClientAliveCountMax 超时次数

用户权限

什么是用户权限？

就是用户的权利，即用一个帐户登录后,有些功能可以使用，有些功能无法使用，这就是管理员对其设置的权限,只有附合权限的人才可以使用对应的功能。权限就是权利的限制范围。

对于用户权限的操作

了解用户管理的文件

/etc/passwd     保存用户信息

 /etc/shdaow     保存用户密码（以加密形式保存）

 /etc/group      保存组信息

 /etc/login.defs   用户属性限制,密码过期时间,密码最大长度等限制

 /etc/default/useradd 显示或更改默认的useradd配置文件

  • 查看用户与用户组

  • 创建用户与用户组

  • 删除用户和用户组

  • 用户和用户组配置文件

查看用户与用户组
cat /etc/passwd 可以查看所有用户的列表
但是cat命令呈现的会比较乱，更换命令
cat /etc/passwd|grep -v nologin|grep -v halt|grep -v shutdown|awk -F":" '{ print $1"|"$3"|"$4 }'|more

w 可以查看当前活跃的用户列表

cat /etc/group 查看用户组
创建用户与用户组（在root权限下操作）
创建用户组
groupadd 选项 用户组
- -g GID 指定新用户组的组标识号（GID）。
- -o 一般与-g选项同时使用，表示新用户组的GID可以与系统已有用户组的GID相同

创建用户
useradd 选项 用户名
- -c comment 指定一段注释性描述。
- -d 目录 指定用户主目录，如果此目录不存在，则同时使用-m选项，可以创建主目录。
- -g 用户组 指定用户所属的用户组。
- -G 用户组，用户组 指定用户所属的附加组。
- -s Shell文件 指定用户的登录Shell。
- -u 用户号 指定用户的用户号，如果同时有-o
删除用户与用户组
删除用户
userdel 选项 用户名
-f：强制删除用户，即使用户当前已登录;
-r：删除用户的同时，删除与用户相关的所有文件

删除用户组
groupdel 用户组
用户和用户组的配置

相关文件配置

/etc/group 注：用户组（group）配置文件
/etc/gshadow 注：用户组（group）的影子文件

1.UID设置

只允许root的UID为0

cat /etc/passwd 命令可进行查看

检测命令：awk -F “：” '($3==0)  {print  $1} ' /etc/passwd

2.禁用无用的账号

awk -F':' '{ print $1}' /etc/passwd或cut -d: -f1 /etc/passwd

检查是否存在无用的账号

passwd -l user 锁定不必要的账号

3.用户密码强弱的检查

john /etc/shadow --single

john /etc/shadow --wordlist=pass.dic

awk -F: '($2=="")' /etc/shadow 查看空口令账号

加固方式：passwd 用户名 为空口令的账号设置密码

4.用户组权限

授予用户对peter.net目录拥有rwx权限

chmod  –R  u+rwx  peter.net

授予组对peter.net目录拥有rwx权限

chmod  –R  g+rwx  peter.net

授予用户、组、其他人对jpeter.net目录拥有rwx权限

chmod  –R  u+rwx,g+rwx,o+rwx  peter.net

撤销用户对peter.net目录拥有w权限

chmod  –R  u-w  peter.net

撤销用户、组、其他人对peter.net目录拥有x权限

chmod  –R  u-x,g-x,o-x peter.net

授予用户、组、其他人对jpeter.net目录只有rx权限

chmod  –R  u=rx,g=rx,o=rx  peter.net

5.对于组和用户的详细命令

groupadd用法

-f, --force            如果组已经存在则成功退出；

并且如果 GID 已经存在则取消 –g；

-g, --gid GID             为新组使用 GID；

-h, --help                显示此帮助信息并推出；

-K, --key KEY=VALUE       不使用 /etc/login.defs 中的默认值；

-o, --non-unique          允许创建有重复 GID 的组；

-p, --password PASSWORD   为新组使用此加密过的密码；

-r, --system              创建一个系统账户；

groupmod用法

-g, --gid GID             将组 ID 改为 GID；

-h, --help                显示此帮助信息并推出；

-n, --new-name NEW_GROUP 改名为 NEW_GROUP；

-o, --non-unique          允许使用重复的 GID；

-p, --password PASSWORD   将密码更改为(加密过的) PASSWORD；

groupdel用法

groupdel admin                 删除admin组；

练习

1.用户登录服务器系统时，应该有警告标题“warning”内容为“Forauthorized users only”提示信息;

2.一分钟内仅允许 3 次登录失败的尝试，超过 3 次，登录帐号锁定 2 分

3.远程用户非活动会话连接超时应小于等于 9分钟，且超时次数不超过3次

数据库加固

限制MySQL用户运行的权限

编辑文件

vi /etc/my.cnf

文件解析

1.启动mysql服务进程的用户

xxx=mysql

2.服务端口号

默认：3306

MySQL创建用户

CREATE USER 'username'@'localhost' IDENTIFIED BY 'password';

3.进入mysql

mysql -u xxx -p

密码9v7TdCgCl1%8

查看所有库

show databases；

删除表 drop

改变默认Mysql管理员用户

update user set user='xxx' where user='root';
4.使用 MySql 内置 MD5 加密函数加密用户的密码
update user set password=md5(password) where user="xxx";
5.赋予用户对数据库所有表只有 select,insert,delete,update 权限
grant select,insert,delete,update on testdb.* to xxx@‘localhost’;

权限检验

show grants for '用户'@'localhost';

重置管理员密码

进入/etc/my.cnf

添加skip-grant-tables

回到mysql

update mysql.user set password=password('密码') where user='管理员名称';
6.禁止MySQL对本地文件进行存取

进入/etc/my.cnf

添加以下代码

set-variable=local-infile=0

保存即可

7.禁止使用 MySql 命令行历史记录
rm -f $HOME/.mysql_history
ln -s /dev/null $HOME/.mysql_history
8.开启 MySql 二进制日志
vi /etc/my.cnf

log-bin=mysql-bin
server mysql restart  #重启数据库

练习

1.以普通帐户 MySql 安全运行 MySql，禁止 MySql 以管理员帐号权限运行

2.删除默认库

3.改变默认 MySql 管理员用户为:sproot

4.使用 MySql 内置 MD5 加密函数加密用户 user1 的密码为（Pa$$w0rd）

5.赋予 user1用户对数据库所有表只有 select,insert,delete,update 权限

6.对忘记 mysql 数据库 sproot 管理员密码进行重置操作

7.禁用 LOCAL INFILE 命令防止非授权用户访问本地文件

8.为防止数据库操作信息泄露,请禁止使用 MySql 命令行历史记录

9.为了分析数据库运行过程中的一些异常活动,请开启 MySql 二进制日志
